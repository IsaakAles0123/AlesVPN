Если в историю Git когда-то попали секреты (.payments.sqlite, .env, ключи)

Вариант A — git-filter-repo (рекомендуется)
-------------------------------------------
Установка (Windows): pip install git-filter-repo

Из корня репозитория (СДЕЛАЙ РЕЗЕРВНУЮ КОПИЮ ПАПКИ ИЛИ КЛОН):

  git filter-repo --path .payments.sqlite --invert-paths

Или убрать несколько путей — см. документацию git-filter-repo.

После переписывания истории нужен принудительный push (осторожно, если есть другие разработчики):

  git push origin master --force

Всем остальным: заново clone или git fetch + reset.

Вариант B — BFG Repo-Cleaner
-----------------------------
См. https://rtyley.github.io/bfg-repo-cleaner/

После любой очистки истории
----------------------------
- Считайте утёкшие секреты скомпрометированными: смените YOOKASSA ключи, BOT_TOKEN,
  PAY_WEBHOOK_TOKEN, пароли БД, при необходимости перевыпустите клиентские ключи WG.

Если репозиторий приватный и секреты не пушились — этот файл можно игнорировать.
